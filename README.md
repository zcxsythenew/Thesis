Linux：运行 `src/artratex.sh xa`

## 论文评阅意见

### 专家1

该论文针对容器化环境下的横向移动攻击检测面临的检测特征不明确、任务分类难度大等问题，分别从网络流量特征分析、横向移动时空特征建模以及横向移动检测方法上提出了相应的解决方案，有效提升整体检测效率。该论文选题具有一定的应用价值和创新性。但该论文仍然存在以下的不足之处：

1. 建立论文采用多种数据集进行交叉验证；
2. 在特征重要度评估部分，需要对其所提方法准确性进行验证，同时尽可能与其他已有方案进行对比评估；
3. 在横向移动时空特征分析部分，建议对参数配置、实验结果进行详细的解释分析原因，而不是简单的看图说话；
4. 对于移动检测方法部分，建议论文增加对已有其他方案的效果对比以突出自身的先进性；
5. 该论文仍有部分段落文字错误，如摘要存在重复语句，论文需要整体进行认真修改；
6. 该论文排版存在一些问题，页面出现大面积空白（如p37，p44），需要整体挑战论文排版。

写作相对规范，问题发现，解决思路输出，挑战引出，具体方法陈述，以及相关验证与评估等等，全文逻辑清晰，能够将所做工作清楚表达出来。

## 评阅后修改情况说明

### 专家1评阅意见及修改情况

1 **建议论文采用多种数据集进行交叉验证**

修改情况：

在论文写作过程中，曾考虑过采用多种数据集进行交叉验证，但由于缺乏可用的公开数据集，最终仅有Kubernetes-dataset数据集最适用。

收到评阅意见后，重新调研了可用的数据集情况。Flora等人 总结了近年来可用的入侵检测数据集，其中与容器环境相关的数据集仅有Kubernetes-dataset、CB-DS 和LID-DS ，但CB-DS和LID-DS均记录单台主机上的系统调用信息，而不是容器集群中的网络流量信息。单台主机上的系统调用信息无法适用于由多台主机构成的容器集群；此外，Sever等人 指出，基于网络流量的方法比基于系统调用的方法具有更高的性能。

另一方面，与网络流量相关的数据集有CIC-IDS-2017 和CSE-CIC-IDS-2018 ，但这些数据集一是并不在容器化场景下，二是没有针对横向移动场景来研究。这些数据集在一个企业网络环境中收集，包含SQL注入、暴力攻击、心脏出血等攻击场景，但不包含具有侦察、立足、横向移动等阶段的完整的攻击链。

上述有关数据集的调研情况已补充到论文的第2.4节。此外，由于目前的高质量数据集仍然十分缺乏，因此调整了论文的总结与展望部分，未来展望增加在真实环境中采集数据集的工作。

2 **特征重要度评估部分**

需要对其所提方法准确性进行验证，同时尽可能与其他已有方案进行对比评估。

修改情况：

对于特征重要度，论文第5.6.6节已对基于特征重要度的特征筛选性能进行了评估，在保留57个特征时模型达到最佳性能，AUC分数达到0.9640；使用全部79个特征时，AUC分数为0.9615，验证了特征重要度评估的有效性。

收到评审意见后，在第3.6节增加了使用GRU模型进行特征重要度验证，对横向移动流量和良性流量进行预测并按照预测误差评分，对比其AUC指标。实验使用了全量特征（79维）和压缩后的特征（57维）进行实验，并与另一种特征降维方法PCA进行对比评估。结果表明，使用本文提出的方法进行降维，去除无关特征后，模型的性能有所提高；使用PCA进行降维，模型性能反而下降，并且PCA丢失了降维方法的可解释性。

3 **横向移动时空特征分析部分**

建议对参数配置、实验结果进行详细的解释分析原因，而不是简单的看图说话。

修改情况：

增加了对参数配置的解释和对实验结果的分析。对于空间特征嵌入实验，解释了随机游走算法Global和Local的设计目的，指出Global方法从更加宏观、全局的角度可以更稳定地学习图的结构；阐述了深度图神经网络方法具有更大的容量，有更强的表达能力，因此其收敛速度快，并且性能更佳。对于时间特征嵌入实验，阐述了三层模型由于一层模型、LSTM优于GRU的原因同样是具有更大的容量，因此选用了三层LSTM用于后续研究。

4 **移动检测方法部分**

建议论文增加对已有其他方案的效果对比以突出自身的先进性。

修改情况：

论文第5.6.5节已对本文提出的方法和其他方案进行了对比。本文提出的方法在TPR、F1、AUC等指标上均优于其他方案，并在FPR上也优于除Bowman等人以外的其他方案（Bowman等人的方案检测率很低）。此外，还通过ROC曲线图，详细描述了论文提出方案的先进性，该方案在提高检测率、降低误报率两个方面都优于其他方案。

收到评审意见后，在论文第5.6.5节，补充了不同方法的时间开销数据及其分析。在TGN、USAD、OmniAnomaly、TranAD和本文提出的方法之间，本文提出的方法时间开销最低，这是因为与TGN相比，本文没有将时序信息与空间结构耦合在一起训练；与USAD、OmniAnomaly、TranAD相比，本文去除了它们采用的对抗训练方式，因此缩短了训练时间。

另外，在论文第5.6.9节的集成验证部分，增加对总体集成方法和其他方案进行效果对比，突出论文提出的方法能够检测出关键的横向移动行为。不论是采用其他方案还是仅采用论文提出的第二阶段的方案，均不能有效检出关键的横向移动流量；只有采用论文提出的两阶段集成的横向移动检测方案才有效，从而验证了论文所提出的方法的有效性和先进性。

5 **文字错误**

该论文仍有部分段落文字错误，如摘要存在重复语句，论文需要整体进行认真修改。

修改情况：

已修改了摘要中和论文其他地方的文字错误。如，删除了摘要中的重复语句；第1.1节修改了部分有语病的句子；第2.1.1节“内容”修改为“资源”；第2.2.3节，增加说明“大多数横向移动检测方法通过网络流量来检测横向移动，但它们仅关注由登录活动产生的网络流量”；第5章修改小节标题，使两个阶段更加清晰；第5.6.7节消融实验中，去除两个模块后，ROC曲线“上移”改为“下移”；修改了第6章的结构；修改了其他个别文字错误。

6 **排版问题**

该论文排版存在一些问题，页面出现大面积空白（如p37，p44），需要整体调整论文排版。

2 **评价指标问题**

## 答辩问题和回答摘要

问：你的工作是一个事后溯源的工作还是在线检测的工作？

答：是在线检测工作，离线训练之后可以部署在线进行检测。

问：有没有其他评估指标，例如攻击发生之后多久可以检测出来？

答：目前没有此类评估指标，因为模型的准确性还有继续提升的空间，误报率还不够低，尚未达到测量此类指标的程度。

问：论文部分图片不清晰。

答：这些图片会在答辩后进行修改。

问：需要其他数据集进行交叉验证。

答：目前没有其他公开的数据集可用，需要进行数据采集的工作，这将作为未来的工作方向。

问：论文中一些模块或机制的原理阐述还不够清晰。

答：这部分内容会在答辩后补充完善。

问：希望你在考虑机器学习各模块的过程中，多思考，因为不同的模块都会起到不同的作用；而不是仅仅通过实验，发现哪个好用就用哪个。

答：感谢老师的建议，未来会在这方面继续加强。
